Executes the stage if the current build is for a change request (a.k.a. Pull Request on GitHub and Bitbucket, Merge Request on GitLab or Change in Gerrit etc.).

Similar as for branch and tag it checks the values of the CHANGE_* environment variables set by the branch-api plugin, you can check the values of those via individual parameters. If more than one parameter is specified, then all of the parameters must match.

Using no parameters i.e. when { buildingChange() } it will just check that CHANGE_ID is available and has a value.

Parameters:

id
CHANGE_ID using ANT style pattern.

target
CHANGE_TARGET using ANT style pattern. The name of the target branch.

branch
CHANGE_BRANCH using ANT style pattern. The name of the actual head on the source control system which may or may not be different from BRANCH_NAME. For example in GitHub or Bitbucket this would have the name of the origin branch whereas BRANCH_NAME would be something like PR-24.

fork
CHANGE_FORK using ANT style pattern. The name of the fork if the change originates from one.

urlX
CHANGE_URL using Regular Expression. The url to the change.

title
CHANGE_TITLE using ANT style pattern.

titleX
CHANGE_TITLE using Regular Expression.

author
CHANGE_AUTHOR using ANT style pattern.

authorX
CHANGE_AUTHOR using Regular Expression.

authorDisplayName
CHANGE_AUTHOR_DISPLAY_NAME using ANT style pattern.

authorDisplayNameX
CHANGE_AUTHOR_DISPLAY_NAME using Regular Expression.

authorEmail
CHANGE_AUTHOR_EMAIL using ANT style pattern.

authorEmailX
CHANGE_AUTHOR_EMAIL using Regular Expression.
