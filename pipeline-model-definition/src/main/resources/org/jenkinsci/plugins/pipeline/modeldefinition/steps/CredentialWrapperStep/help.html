Specify a credentials id to be a part of the pipeline environment in a slightly opinionated way.

This is intended to be used in declarative pipelines, see example below.

In the below example, if the credentials with id my-login-id is a Standard Username and Password the environment would be populated with three variables: LOGIN, LOGIN_USR and LOGIN_PSW. If the credentials is for example a secret text, the variable LOGIN would contain that secret text. Etc. 

    pipeline {
        agent any
        environment {
            LOGIN = credentials 'my-login-id'
        }
    }

Advanced:
The step can be used in standard pipeline scripts where it returns an object that can generate the parameters to send to the withCredentials step. To do that either call List params = c.resolveParameters("MY_VAR") or c.addParameters("MY_VAR", myExistingList)
